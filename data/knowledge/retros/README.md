# Retros de Lanzamientos

Cada retro se genera automaticamente con el comando `/cerrar-lanzamiento {id}`.

## Formato de archivo
`{YYYY-MM}-{producto}.md`

## Estructura de una retro
- Producto, tipo de funnel, ticket, fechas
- Numeros reales: leads, ventas, revenue, conversion, CPA, ROAS
- Que funciono (top 3)
- Que fallo (top 3)
- Que cambiar la proxima vez (top 3)
- Insights no obvios

## Como se usa
- El estratega lee las 2-3 retros mas relevantes al iniciar un lanzamiento nuevo
- Relevancia = mismo tipo de funnel, ticket similar, o mismo mercado
- Las retros son el activo mas valioso del sistema: conocimiento que crece con cada lanzamiento

> **IMPORTANTE**: Escribe aqui retroactivamente 1-2 retros de lanzamientos pasados para arrancar con knowledge real.
